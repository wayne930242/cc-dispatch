## Detailed Findings

### `CLAUDE.md` — Needs Fix

**Major Issues:**

- **Line 22 — Linter-enforceable redundancy.** The Verification commands section lists `go vet ./...` as a command to run, but the PostToolUse hook at `.claude/hooks/go_format_vet.py` already runs `go vet` on every `*.go` Write/Edit automatically. Listing it as a manual verification step duplicates deterministic enforcement.
  **Suggested fix:** Remove `go vet ./...` from the Verification commands list (keep `go test ./...` and `go build ./cmd/ccd`).

- **Line 31 — Duplicates daemon-stability rule.** The "Stability boundary" bullet restates what `.claude/rules/daemon-stability.md` (auto-injected when reading `internal/daemon/**`, `internal/mcp/**`, `internal/db/**`, `internal/client/**`) already says. The summary will drift from the authoritative rule over time.
  **Suggested fix:** Remove the bullet entirely. Path-scoped rules auto-inject; no CLAUDE.md pointer is needed.

- **Line 34 — "Never stage it" duplicates global git-safety.** The clause overlaps with `~/.claude/rules/git-safety.md` ("stage specific files by name"). Project-specific auth-token warning is fine; the staging clause is redundant.
  **Suggested fix:** Keep the file identification and "never echo in logs" warning; drop "Never stage it."

**Minor Issues:**

- **Line 37 — Stale commit hash reference.** `(see commit 49e6d99)` anchors the rule to a commit that may be squashed or rebased. Remove the parenthetical.

- **Line 41 — Traditional Chinese line duplicates user global.** `~/.claude/CLAUDE.md` already contains "Never use Simplified Chinese. Always use Traditional Chinese for all communication." The project-level line adds no new information and consumes tokens every session.
  **Suggested fix:** Remove the Communication section (or drop this line and keep only project-unique communication conventions).

### `.claude/skills/releasing/SKILL.md` — Needs Fix

**Major Issues:**

- **Line 6 — `allowed-tools` missing `Write`.** Task 2 step 1 says "if missing, create with a `# Changelog` header and a `## [Unreleased]` section." Creating a new file requires the Write tool, which is not in `allowed-tools`. With `disable-model-invocation: true`, permission already needs to flow through `allowed-tools`; Edit cannot create files.
  **Suggested fix:** Change `allowed-tools` to `Bash(git *) Bash(gh *) Read Write Edit Grep Glob`.

- **Line 3 — Description enumerates workflow.** The description begins with "Use when..." (correct) but then lists the workflow steps: "cutting a new version tag, updating CHANGELOG, aligning README install instructions, and scanning for leaked secrets before tagging." Workflow summary in the description causes Claude to read the description instead of loading the body.
  **Suggested fix:** Rewrite to describe only triggers, e.g. `description: Use when the user asks to prepare, cut, ship, or tag a cc-dispatch release. Trigger phrases include "prepare release", "cut release", "tag v0.X", "new version", "ship a release", "/releasing". Argument is the target version (e.g. v0.2.0).`
